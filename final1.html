Final Exam Prep#

Introduction#

  • Following are a set of questions that may or may not show up in a similar form on the exam

  • Feel free to discuss these on the Piazza site

  • The exam will be closed-book, closed-notes, closed-computer and held in class.

How to do well on the exam#

  • The exam will be on April 30th. Spend a small amount of time each week answering the questions below. A smaller, but sustained effort will help you absorb the material better and keep stress to a minimum.

  • Ask questions on Piazza! Share and discuss answers!

Questions - Inter-process Communication#

  1. List the following forms of IPC from faster performance to slower performance: files, named pipes and memory mapped files

  2. What guarantees does a named-pipe give you?

  3. What guarantee do I lose if I send too much data at once over a named-pipe?

  4. Is it safe to use pointers in memory mapped regions? Why?

  5. Re-write these structs so that they’ll work properly with memory mapped files:

Questions - Virtual Memory#

  1. What is Virtual Memory responsible for?

  2. How does Virtual Memory help a system to perform better?

  3. What is a physical address? What is a virtual address?

  4. What role does the MMU serve?

  5. What role does the page table serve?

  6. How does a two or three level page table save memory? What kind of data structure is it?

  7. How does a page fault occur?

  8. What are the four classifications of pages as used by a page replacement algorithm?

  9. Which of the two algorithms is subject to Belady’s Anomaly - NFU or FIFO

  10. If a page replacement algorithm is not subject to Belady’s Anomoly, what does this imply when more physical memory is added to a system?

  11. If I have the following distance string - 4, 4, 1, inf., 7, 2, 2. What is the page fault rate for a physical memory size of 2? For 5? (See 8.35 in the notes)

  12. What is a working set?

  13. What is the difference between local and global paging?

  14. Why does DMA need its pages to be locked?

  15. Explain how COW (Copy on Write) works when a program calls fork() and then writes to a data structure.

  16. List the steps in the process of a page fault (8.51, 52)

Questions - Storage and Devices#

  1. Is a hard disk a character or a block device?

  2. What is RAID 0?

  3. What is RAID 1?

  4. What is RAID 5?

  5. Explain how to calculate the parity of two bytes into a third byte.

  6. What does parity have to do with RAID 5?

  7. What is the policy of the Shortest Seek First disk scheduling algorithm?

  8. What is the down side of Shortest Seek First?

  9. What is the policy of the Elevator Algorithm?

  10. Which part of the disk does the Elevator Algorithm favor in terms of average response time?

  11. How does FSCAN modify Shortest Seek First to get rid of its disadvantage?

Questions - Implementing Files and Folders#

  1. Linked lists are used to map files onto some sequence of blocks in some filesystem implementations. What are the advantages / disadvantages of this?

  2. What change does FAT make from the linked list approach?

  3. What are the advantages to FAT? What are the disadvantages?

  4. Explain what an inode is

  5. Why are several pointers (zones) to the first few data blocks stored in the first inode of the Minix inode structure? What benefit does this give?

  6. Explain what an indirect and double indirect zone are. Draw a diagram.

  7. Why are block caches needed? What do they help?

  8. What responsibilities does a VFS (Virtual Filesystem) manage?

  9. What is FUSE? How does VFS enable it?
